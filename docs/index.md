## 为什么
首先，看到知乎文章[为什么中国学生这么忙但是实际没有学到多少](https://www.zhihu.com/answer/2919982157)有感。先谈谈文中提到的现象吧。我可能在学习的大多数时候属于那个Z，事无巨细地摘抄大小知识点，整整齐齐记在笔记本，完全不理解。不理解倒是其次，关键是不知道自己在学什么：不停地记，浪费了时间；不停地学，知识它不进脑子。记笔记有用吗？应该是有用的，写过一遍总会留下印象。对于学习效率和效果的帮助打吗？上述记笔记方式绝对不大。这种机械记录和记忆说到底是把 “每个字都认识，连起来就懵逼” 的一串字符写在可存储媒介上，就好像每天在街上擦肩而过的陌生人一样，转瞬即忘。**我尚未找到一个合适的学习方法**， 由这篇文章的褒贬态度，同时我也很赞同的方法是在实践中找寻自己的不足，在一次磕磕碰碰的作业中寻找漏洞，寻找方法。要举例就是这次搭建github page，以前对于git的理解停滞不前，记不住`pull`, `push`, `commit`, `add`这些指令的作用和用法，也不明白git究竟是怎么提交的。但实际应用中绝大多数项目的更新操作都是通过git提交，那么一个我仅仅通过模仿教程搭建一个github这样简单的小项目就大量地练习了git的基本使用也是合理的了。重复多次以及教程提到的一些理解让我明白首先`git add`相当于暂存当前版本，然后`git commit`提交，最好附带一些描述信息，最后`git push`完成push到远程仓库的操作。git教程中自然也有上述内容，但总不会比实践来的印象深刻。

其实这种现象在大学应该是更明显的：一学期开了各种杂七杂八的课，课程兼具迷惑性和紧迫性，指学习内容多且复杂，学时少期末赶，学的时候惶恐不安，考试发现年年都是这几样。在你左职的培养方案里，理工科大一的线代只有32学时，闭着眼过完整本书发现只会矩阵乘法、行列式、特征值balabala，连很重要的线性变换都没有讲，说到底都是走个形式。可能是因为教学内容有上级的要求，实际学时又不能太多，因此绝大多数课都会表现为学习难，考试简单（简单是指考的知识点集中，不会像高中那样覆盖整本书）。把这点搞清楚，也许至少大学的考试可以有准备地应付了。当然我还没有来得及实践，毕竟第三学期考的也是一坨稀烂。
